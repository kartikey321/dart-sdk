e` for that path.

To optimize read times, this resolved state is **pre-computed and natively flattened** into the employee's Firestore document (under `centralAccess` or the project's `access` array). Validating permissions at runtime requires no complex join logic; it is simply a direct traversal of the pre-computed tree.

### 3. Deny-By-Default Paradigm

Permission verification algorithms should execute as follows:

1. Target the correct payload context: Company-level vs Project-level.
2. Traverse the pre-computed tree: `Module(moduleKey) -> SubModule(subModuleKey) -> AccessValue(accessKey)`.
3. If the path leads to a boolean `value`, return that `value`.
4. If the path is broken, does not exist, or the key is absent, immediately return **`false`**.

This strict **deny-by-default** stance ensures maximum security.
